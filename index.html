課程大綱

60 分鐘學會什麼？

四個核心模組，從觀念到實用方法，一次搞懂 Rovo

1

Prompt 基礎心法讓 AI 聽懂你在說什麼

掌握與 AI 溝通的核心技巧，從此告別「AI 聽不懂我」的困擾。

  • 什麼是 Prompt？為什麼「問法」決定「答案品質」？
  • 好 Prompt 的三要素：角色（Role）、任務（Job）、背景（Context）
  • 可選元素：格式、語氣、來源、範例
  • 迭代思維：用後續提示優化結果
2

Rovo Search跨平台智慧搜尋

一個入口，找到所有答案。整合企業內所有知識來源。

  • 整合 Atlassian Cloud + 第三方平台（Slack、Google Drive、Figma、GitHub⋯）
  • 搜尋書籤（Bookmarks）：管理員設定常見問題的「秒答」捷徑
  • 知識卡片（Knowledge Cards）：自動彙整專案、術語、流程定義
  • 實戰場景：企業知識管理、跨團隊協作、客服支援
3

Rovo Chat對話式知識存取

像問同事一樣問 AI，直接查詢組織內部知識。

  • 透過對話介面，直接查詢組織內部知識
  • @ 提及人員/團隊、貼上 Smart Link 精確指定檔案
  • Definition 功能：滑鼠懸停即顯示術語定義
  • 實戰應用：會議議程生成、簡報大綱撰寫、Onboarding 文件整理
4

Rovo Agent你的 AI 虛擬隊友

不只回答問題，還能幫你執行任務。

  • 開箱即用的官方 Agent：Issue Organizer、Release Notes Drafter
  • Progress Tracker（專案進度追蹤）、Product Requirements Expert（PRD 審查）
  • Studio 簡介：用自然語言打造專屬 Agent（概念介紹）
  • 100+ Agent Skills：建立頁面、建立 Issue、轉換狀態、加入 Sprint

課程大綱

你會學到什麼？

5 個實作練習，從建立到優化，完整掌握 Agent 開發流程

1

建立你的第一個 AgentCreate the Agent

從零開始建立 Bug Review Agent，學會撰寫有效的 Instructions。

  • Agent 建立流程與介面操作
  • Instructions 四大要素：角色、目標、人設、輸出格式
  • 設定 Conversation Starters
  • 測試與驗證 Agent 行為
2

優化 Agent 輸出Improve Your Agent

學會控制 Agent 輸出，移除冗餘訊息，只保留有價值的內容。

  • 移除開頭與結尾的冗餘訊息
  • 避免重複 Summary 與 Description
  • 讓 Agent 只輸出改善建議
3

新增 Action 功能Add an Action

讓 Agent 不只能回答，還能直接執行動作。

  • 認識 Agent Actions
  • 新增「Add comment to work item」功能
  • 實作：讓 Agent 自動將建議寫入 Jira Comment
4

處理例外情境Acknowledge Good Bug Reports

讓 Agent 學會判斷，遇到優質報告時給出不同回應。

  • 問題分析：Agent 總是有話要說
  • 運用 Agent 自我優化 Instructions
  • 迭代調整技巧與除錯方法
5

結合 AutomationTry Out the Automation

將 Agent 與 Jira Automation 結合，實現自動化工作流程。

  • 使用 AI 輔助建立 Automation Rule
  • 設定觸發條件與 Agent 呼叫
  • 條件判斷與自動 Comment

你的痛點

這堂課幫你解決什麼問題？

🏷️

Issue Type 常被選錯

使用者提交 Bug 時常選錯類型，需要人工重新分類，浪費時間。

審查流程太繁瑣

每次要檢查報告品質、提取環境資訊、指派負責人⋯⋯步驟太多。

📋

環境資訊散落在描述中

報告者把 OS、瀏覽器版本寫在描述裡，沒有填到專用欄位。

🤝

單一 Agent 能力有限

一個 Agent 做不了所有事，需要多個 Agent 分工合作。

課程大綱

你會學到什麼？

建立 3 個 Agent，串接成完整的自動化工作流程

1

建立 Bug Identifier AgentBuild a Bug Identifier Agent

建立能自動識別 Issue 是否為 Bug 的 Agent。

  • 使用 AI 輔助建立 Agent（不用手動設定）
  • 設計精準的輸出格式：「bug」或「no bug」
  • 學會如何讓 Agent 只輸出你要的內容
2

建立 Field Analyzer AgentCreate a Field Analyzer Agent

建立能從描述中提取環境資訊的 Agent。

  • 分析描述中的 OS、瀏覽器、裝置資訊
  • 結構化輸出環境變數
  • 處理「無資訊」的例外情境
3

建立多 Agent 工作流程Building a Multi-Agent Workflow

將三個 Agent 串接成完整的自動化流程。

  • 使用 {{agentResponse}} 傳遞數據
  • 設計條件分支邏輯（IF / ELSE IF）
  • 自動變更 Issue Type、新增 Comment、指派負責人
  • 自動填寫 Environment 欄位

自動化整合實戰課程

學會讓 Agent 輸出 JSON 格式，結合 Automation 批次建立工作項目，將 Epic 自動拆解為多個 User Stories。

你的痛點

這堂課幫你解決什麼問題？

📦

拆解 Epic 太花時間

每次都要手動分析 Epic、想 User Stories、一個個建立，耗時又容易遺漏。

📝

Agent 輸出格式不統一

Agent 給的建議是純文字，沒辦法直接被 Automation 解析使用。

🔗

無法批次建立工作項目

希望一次建立多個 User Stories，並自動關聯到 Epic。

⚡

想要更智慧的自動化

讓 AI 分析需求、自動產出 Stories，而不只是機械式的流程自動化。

課程大綱

你會學到什麼？

掌握 JSON 輸出與批次建立技巧，實現智慧化工作項目生成

1

建立 Story Builder AgentBuild an Epic-to-User-Stories Agent

建立能分析 Epic 並建議 User Stories 的 Agent。

  • 分析 Epic Description 並提取需求
  • 設計 TEXT 格式輸出：Story / Summary / Description
  • 使用「As a... I want to... So I can...」格式
2

設定 JSON 格式輸出Output in JSON Format

讓 Agent 能根據需求輸出 TEXT 或 JSON 格式。

  • 設計雙輸出模式：[TEXT FORMAT] / [JSON FORMAT]
  • 撰寫 JSON Schema 規範
  • 確保輸出為有效的 JSON 格式
  • 測試與驗證 JSON 輸出
3

批次建立 User StoriesCreate User Stories Automatically

結合 Automation 實現批次建立工作項目。

  • 使用 {{{agentResponse.aslist}}} 解析 JSON 陣列
  • 設定 Advanced Branching 迴圈處理
  • 自動建立 Stories 並關聯到 Epic
  • 使用 {{userstory.Summary}} 動態取值
